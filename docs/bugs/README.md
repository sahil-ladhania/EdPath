# docs/bugs

Issues found **during the build** — tracked here so they aren't lost between sessions. Empty until the build surfaces real bugs; do not pre-populate.

- [`known-issues.md`](./known-issues.md) is the running log.
- One entry per issue. Suggested format:
  - **ID / date** — short title
  - **Symptom** — what's observed
  - **Where** — file / node / layer (e.g. `N4 await_input`, `apps/edpath-web/...`)
  - **Status** — open / investigating / fixed
  - **Fix / PR** — link or commit once resolved

Keep it factual. Link related decisions (`../decisions/`) or handoff notes (`../handoff/`) where relevant.
